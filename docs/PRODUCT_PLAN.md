# Y Music Spectrogram - 产品需求文档 (PRD)

> 版本: v1.0 | 日期: 2026-04-04 | 作者: PM

## 一、产品现状分析

### 当前能力
- 通过 ScreenCaptureKit 捕获系统音频，vDSP FFT 实时分析
- 菜单栏 150pt 宽度频谱柱状图可视化
- 5 种颜色方案（Rainbow / Green-Red / Blue-Red / Monochrome / Custom）
- 频带数量可调（8-64）
- 锁屏后自动恢复捕获
- 基本设置窗口（频带数、配色）

### 核心问题清单

| # | 问题 | 严重程度 | 说明 |
|---|------|---------|------|
| 1 | 设置窗口 UI 粗糙 | 高 | 无分组、无图标、纯文字堆叠，不符合 macOS 设计规范 |
| 2 | 菜单全英文 | 高 | 目标用户为中文用户，所有 UI 文字应中文化 |
| 3 | 无错误反馈 | 高 | 权限被拒、捕获失败时用户无感知，只有 print 日志 |
| 4 | 菜单项缺少状态 | 中 | Start/Stop 同时显示，不反映当前捕获状态 |
| 5 | 无"关于"窗口 | 中 | 缺少应用信息、版本号展示 |
| 6 | 无开机自启动 | 中 | 用户每次重启需手动打开 |
| 7 | 设置项不足 | 中 | 缺少灵敏度调节、柱宽间距等可视化细调 |
| 8 | 视觉效果基础 | 低 | 柱状图缺少动效打磨，可提升视觉吸引力 |

## 二、第一轮迭代需求（本次实施）

### 目标
将 demo 级应用提升为可日常使用的完整产品，聚焦 **P0 核心体验** 和部分 **P1 增强功能**。

---

### P0 - 必须完成

#### P0-1: 设置窗口重设计

**目标**: 符合 macOS 原生设计规范的精美设置界面

**具体要求**:
- 使用 macOS 标准 `Form` + `Section` 分组布局
- 分为以下区域:
  1. **显示设置**: 频带数量滑块、柱状图间距调节（1-4pt）、灵敏度调节（低/中/高）
  2. **配色方案**: 颜色方案选择器 + 实时预览条（在设置中嵌入一个小的频谱预览）
  3. **通用设置**: 开机自启动开关
- 窗口标题改为中文「频谱设置」
- 窗口尺寸: 480 x 520，居中显示
- 自定义颜色区域展开/收起应有平滑过渡动画

#### P0-2: 菜单栏右键菜单增强

**目标**: 信息丰富、状态清晰的菜单

**具体要求**:
- 菜单项中文化:
  - 「频谱设置...」(⌘,)
  - 分隔线
  - 「开始捕获」/「停止捕获」— 根据当前状态只显示一个，带状态图标（绿点/红点）
  - 分隔线
  - 「关于 Y Music Spectrogram」
  - 「退出」(⌘Q)
- 菜单项带有对应的系统图标（SF Symbols）

#### P0-3: 关于窗口

**目标**: 展示应用基本信息

**具体要求**:
- 应用图标（使用 SF Symbol `waveform.circle.fill` 作为占位）
- 应用名称: Y Music Spectrogram
- 版本号: 从 Bundle 读取（Info.plist 中配置）
- 简短描述: 「macOS 菜单栏实时音频频谱可视化」
- 版权信息
- 窗口尺寸: 300 x 200，居中

#### P0-4: 错误处理与用户反馈

**目标**: 关键错误路径给用户明确反馈

**具体要求**:
- **权限被拒绝时**: 弹出 NSAlert 提示用户前往系统设置授权，提供「打开系统设置」按钮
- **捕获启动失败时**: 菜单栏显示静态图标（如 `waveform.badge.exclamationmark`）提示异常
- **权限未决时**: 首次启动弹出友好提示说明为什么需要屏幕录制权限
- 错误状态通过回调通知 MenuBarController，由其决定展示方式

#### P0-5: UI 文字中文化

**目标**: 所有用户可见文字使用中文

**具体要求**:
- 菜单项、设置窗口标题和标签、关于窗口、Alert 对话框全部中文
- 日志 print 保持英文（仅开发调试用）

### P1 - 应该完成

#### P1-1: 开机自启动

**目标**: 支持 macOS 登录时自动启动

**具体要求**:
- 使用 `SMAppService.mainApp` (macOS 13+) 实现 Login Item
- 在设置窗口「通用设置」区域添加开关
- 状态通过 SettingsManager 管理
- 切换时即时生效

#### P1-2: 灵敏度调节

**目标**: 用户可调节频谱对音量的响应灵敏度

**具体要求**:
- 在设置中添加灵敏度滑块（0.5x - 2.0x 增益倍数）
- 影响 SpectrumAnalyzer 的 `dbFloor` / `dbCeiling` 范围
- SettingsManager 新增 `sensitivity` 属性持久化
- 默认值: 1.0（当前行为）

#### P1-3: 柱状图间距可调

**目标**: 用户自定义柱状图视觉密度

**具体要求**:
- 在设置中添加间距滑块: 0.5pt - 3pt，步进 0.5
- SettingsManager 新增 `barSpacing` 属性
- SpectrumVisualizerView 读取该设置
- 默认值: 1.0pt（当前值）

## 三、设计规范

### 颜色与风格
- 遵循 macOS 原生设计语言，使用系统控件
- 设置窗口使用 SwiftUI `Form` 风格，自动适配亮/暗模式
- 颜色预览使用圆角矩形色块
- 关于窗口简洁居中

### 字体
- 标题: 系统默认 `.title2`
- 正文: 系统默认 `.body`
- 辅助文字: `.caption`，使用 `.secondary` 颜色

### 动效
- 设置变更即时反映到菜单栏频谱（已有，保持）
- 自定义颜色区域展开/收起使用 `.animation(.easeInOut(duration: 0.2))`

---

## 四、技术约束

1. **不引入外部依赖** — 继续使用系统框架
2. **保持 SPM 构建** — 不转换为 Xcode 项目
3. **最低系统 macOS 13.0** — SMAppService 在 13.0+ 可用
4. **保持现有架构** — 不改变数据流: ScreenCaptureKit → AudioCaptureManager → SpectrumAnalyzer → SpectrumVisualizerView
5. **线程安全** — 新增设置项需遵循现有 NSLock + main queue 模式
6. **Info.plist 变量** — 版本号等通过 `$(VARIABLE)` 占位，由 bundle.sh 替换
